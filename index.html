Flow rules per branch
feat/* — developer iteration
No SonarQube · no build · no deploy · no tests on the branch itself. PR to dev triggers the first SQ scan/Semgrep and DEV build.
dev branch — integration + DEV build
Merge → SQ scan → build DEV package → push GAR-DEV → deploy DEV → smoke tests → DEV gate. Fail = email notification.

PR dev→qa after DEV gate pass → SQ scan on qa branch → build QA package → push GAR-QA → QA lead approval → deploy QA → regression → QA gate.

PR qa→main after QA gate pass → GitHub Ruleset → SQ scan on main → build PROD package → push GAR-PROD → Harness OPA → approval → canary.

Harness OPA policies — PROD only
→
Scan pass: artifact.scanResult.critical == 0 — no CRITICAL CVEs allowed
→
SonarQube gate: sonar.qualityGate.status == "OK" on main branch
→
QA gate on record: qa.testSuite.status == "passed" + qa.approver.count >= 1
→
Image tag format: prod-v[0-9]+.* — "latest", "dev-*", "qa-*" tags blocked in PROD
→
GAR-PROD origin: image must originate from approved GAR-PROD repository only
→
Deploy window: Mon–Thu 10:00–15:00 UTC — holidays blocked, CHG override required
→
Approval counts: DEV=0 (auto) · QA=1 (QA lead) · PROD=1 (Change mgmt)
