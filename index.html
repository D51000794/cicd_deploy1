ServiceNow INC + email on SonarQube gate failure: Any gate failure at DEV build, QA build, or PROD build automatically raises a ServiceNow INC and sends email to the app team. Feature branches are excluded — no SonarQube runs on feat/*. Pipeline blocked until gate passes. Click for full detail.
Each environment has its own build and its own GAR image. A failure at any environment rolls back to the previous image in that environment's GAR repo, then the fix goes back to the dev branch, a new DEV package is built, the fix flows through DEV → PR to QA → QA build → PR to PROD → PROD build. All notifications via ServiceNow INC + email only.

No SonarQube · no build · no deploy · no tests on the branch itself. PR to dev triggers the first SQ scan/Semgrep and DEV build.
